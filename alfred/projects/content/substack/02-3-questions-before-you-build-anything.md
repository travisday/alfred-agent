Subject: 3 questions before you build anything

The first app I built with AI, I ended up deleting 90% of the code.

Not because the code was bad. Because I was building a bunch of features nobody needed — including me.

Building with AI is so fast that it's dangerously easy to keep going. One more feature. One more page. One more integration. Before you know it, you've spent a week on something that should've taken a day, and you still can't explain what it does.

If you're just messing around and learning, go for it. Build whatever.

But if you're trying to build something useful — an MVP to pitch, a tool to save you time, a product to test with real users — you need to answer these 3 questions first.

### 1. What does it do?

Describe it in one sentence.

If you can't do that, the idea isn't clear enough yet. And if you can't describe it clearly, AI definitely can't build it.

"It helps freelancers track unpaid invoices and send reminders." That's clear. That's buildable.

"It's like a dashboard for everything in my business." That's a rabbit hole. You'll build for three days and still not know what the app does.

One sentence. No "and also it does this." Just the core thing.

### 2. Who is it for?

There are really only three answers: you, a specific person you know, or a specific group with a shared problem.

"Everyone" is not an answer.

Building for yourself is massively underrated. No users to worry about. No deployment pressure. Nobody complaining when something breaks. Some of my most useful apps have exactly one user — me.

If you're building for others, you need to be able to describe them specifically. "Small business owners who invoice clients monthly" is specific. "People who want to be more productive" is not.

### 3. What's the smallest version that's useful?

This is the big one.

Take your idea and strip it down. Remove features until you're left with the one thing that makes it actually work.

"Track invoices and send reminders" becomes → "A list of invoices with due dates and a button to send a reminder email."

No analytics dashboard. No client portal. No payment integration. Just the core thing.

You can always add more later. You can never get back the time you spent building stuff that didn't matter.

### Why this matters more with AI

With a traditional developer, scope creep costs you money. Every extra feature means more hours, more invoices, more budget.

With AI, scope creep costs you something worse — it costs you clarity. The app gets bloated, the AI starts making weird decisions because there's too much going on, and you end up with something that technically works but doesn't actually do anything well.

The constraint isn't money anymore. It's focus.

Answer these 3 questions. Write them down. Then build only what they describe.
